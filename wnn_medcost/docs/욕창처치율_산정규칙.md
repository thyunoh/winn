# 욕창 처치를 실시한 환자분율 — 산정규칙 및 사례

> 요양병원 적정성평가 09번 지표. 심평원 공식 산식 기준 정리.
> 최초 작성 2026-07-11 (장정순 사례 계기).

## 1. 지표 개요

- **분류**: 진료영역 과정지표 / **가중치 4**
- **현황값** = 분자 ÷ 분모 × 100
- **분모**: 전월 욕창(1~4단계 중 하나 이상) 보유 + 당월 계속입원 평가(`EVAL_TYPE='2'`) 대상자
- **분자**: 분모 중 **적절한 욕창 처치를 실시**한 것으로 인정되는 환자

## 2. 처치 인정 규칙 (핵심)

### 처치 4항목 (환자평가표 TBL_PATVAL_MST)
| 코드 | 컬럼 | 의미 |
|---|---|---|
| a | `PRESS_REL_DEV` | 압력을 줄여주는 도구 사용 (에어매트리스·에어쿠션·물침대) |
| b | `POS_CHANGE` | 체위변경 (2시간마다, **간호사 직접/주도만** 인정) |
| c | `NUTR_SKIN` | 영양·수분 공급 (의사 처방에 따른 적절한 영양·고단백치료) |
| d | `SKIN_ULC_DRES` | 피부궤양 드레싱 |

### 당월 욕창단계별 인정 요건
| 당월 상태 (PR_ULCER1~4) | 필요 처치 | 근거 |
|---|---|---|
| **유지 (2·3·4단계)** | **a+b+c+d = 4** (드레싱 포함) | 개방창 존재 → 드레싱 필수 |
| **1단계 (`1000`)** | **a+b+c = 3** (드레싱 제외) | 1단계 = 피부 균열 없음 → 드레싱 대상 아님 |
| **완치 (`0000`)** | **a+b+c = 3** (드레싱 제외) | 창상 아물음 → 드레싱 대상 아님 |

> **왜 1단계/완치는 드레싱을 빼는가**
> 심평원 환자평가표 작성 매뉴얼: *"1단계 = 압박 제거 후에도 피부 발적은 있으나 **피부 균열은 없는 경우**"*, 드레싱은 *"실시한 경우에 표시"* 하는 조건부 항목. 개방창이 없으므로 드레싱은 물리적으로 대상이 아님. 단, **압력·체위·영양(a·b·c) 3개는 반드시 모두** 실시해야 인정.

### 전월 단계 × 당월 상태 매핑 (SP 분자 = 그리드 dressingYn='1' 공통)
| 전월 | 당월 | 필요 처치 | 인정 |
|---|---|---|---|
| 있음(전체) | 유지(단계 합>0) | a+b+c+d=4 | ✅ |
| 1단계 | 1단계 | a+b+c=3 | ✅ |
| 2단계 | 1단계 | a+b+c=3 | ✅ |
| **3·4단계** | **1단계** | a+b+c=3 | ✅ (2026-07-11 보정 전 그리드 누락분) |
| 있음(전체) | 완치(0000) | a+b+c=3 | ✅ |
| 위 조건 미충족 | — | — | ❌ 미처치 |

## 3. 구현 위치 (중요)

| 화면 요소 | 산출 주체 | 저장/로드 |
|---|---|---|
| **좌측 지표 분모/분자** | DB 프로시저 `SP_EVALUATION_INDICATORS_CREATE2` | **TBL_WEVALUE_MST 에 "월 자료생성" 시점 저장** |
| 우측 그리드 (해당/미처치) | `Magam_SQL.xml : select_CategoryList09` (dressingYn CASE) | 조회 시 실시간 |
| 화면 렌더/"처치 총" 카운트 | `assessment.jsp` (jobFlag='09') | — |

### ⚠ 반영 시 함정
1. **좌측 분모/분자는 자료생성 시점의 "저장값"** — SP를 고쳐도 **「적정성평가 월 자료생성」을 다시 실행하기 전에는 화면 숫자가 안 바뀐다.**
2. **그리드(select_CategoryList09)는 MyBatis 매퍼 XML** — 수정 후 **WAR 재빌드 + 서버 완전 재시작** 필요 (매퍼는 기동 시 1회 로드·캐시).
3. **Eclipse 빌드 주의**: `.classpath` 에서 `src/main/resources` 가 `excluding="**"` 라 **Eclipse 자체 빌드는 매퍼 XML을 복사하지 않음** → `mvn` 빌드로만 `target/classes` 갱신됨. 로컬 테스트 시 실제 배포 경로 확인 필수.

## 4. 사례 — 장정순 (2026-06, 보성군립요양병원)

### 실제 데이터 (TBL_PATVAL_MST)
| 당월 욕창 | 압력(a) | 체위(b) | 영양(c) | 드레싱(d) | EVAL_TYPE |
|---|---|---|---|---|---|
| `0·0·0·0` (완치) | 1 | 1 | **0** | 0 | 2 |

- `a+b+c = 1+1+0 = 2` → **3 미충족** (영양 미실시)
- 완치 환자이나 **영양공급을 안 했으므로 처치 미인정 = 미처치가 정답**

### 발생했던 오류와 해결
- **증상**: 좌측 지표 분자 **7**, 우측 그리드 처치 **6** (장정순 미처치) → 1명 불일치
- **원인**: 배포된 SP가 "전월 욕창 + 당월 완치"면 **처치항목(영양 등) 충족 여부를 확인하지 않고** 분자에 포함 → 영양 빠진 장정순이 7로 집계
- **조치**: SP 완치/1단계 분자 조건에 `(PRESS_REL_DEV + POS_CHANGE + NUTR_SKIN) = 3` 강제 → **자료생성 재실행**
- **결과**: 분자 **6** 으로 갱신, 현황값 85.71%(6/7), 결과 3.2점. 좌·우·심평원 기준 모두 일치.

### 교훈
- **처치 인정에는 압력·체위·영양 3개가 모두 필요** — 하나라도 빠지면(특히 영양) 완치·1단계여도 미처치.
- **SP 수정 ≠ 화면 반영**. 분모/분자는 저장값이므로 **반드시 월 자료생성 재실행.**
