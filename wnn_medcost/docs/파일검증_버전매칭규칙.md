# 파일 검증 - 버전 매칭 규칙

## 1. 파일 유형별 분류

### 유형 A: 확장자 파일 (CAR, GHP 등)
- **파일 형태**: `파일명.GHP`, `파일명.CAR`, `파일명.B00` 등
- **버전 정보**: 해당 파일 **첫줄 앞 3자리** (`LEFT(lineval, 3)`)
- **SAMFVER 매칭**: `fn_GetJobsSam`으로 확장자 → jobssam 변환 후 매칭
  - GHP 계열: `GHP, B00, B01, C00, C01, B60, B61, C60, C61` → `'GHP'`
  - CAR: `CAR` → `'CAR'`
  - REP 계열: `L32, CXX, L01~L20` → `'REP'`
- **특징**: 파일 하나에 모든 테이블 데이터 포함

### 유형 B: 비확장자 파일 (C/M 계열) - H010 없음
- **파일 패턴**:
  - `M010.1 ~ M020.5`
  - `M010.1 ~ M021.5`
  - `C010 ~ C110.4`
- **버전 정보**: **가장 작은 파일명**의 첫줄 앞 3자리
  - 예: `M010.1`이 가장 작은 파일 → `M010.1` 첫줄 `LEFT(lineval, 3)` = 버전
- **SAMFVER 매칭**: `SAMVER = 파일명 그대로` (예: `'M010.1'`, `'M020.3'`)
- **특징**: 파일 세트로 선택, 각 파일이 개별 테이블에 대응

### 유형 C: 비확장자 파일 (K/D/H 계열) - H010 동반
- **파일 패턴**:
  - `K020.1 ~ K020.4` + `H010`
  - `D020.1 ~ D020.6` + `H010`
  - `H040.1 ~ H040.5` + `H010`
- **버전 정보**: **H010 파일**의 첫줄 앞 3자리
  - `H010`이 항상 함께 선택되며, `H010`이 버전 정보를 가짐
- **SAMFVER 매칭**: `SAMVER = 파일명 그대로` (예: `'K020.1'`, `'H010'`)
- **특징**: H010이 반드시 동반, 파일 세트로 선택

## 2. 버전 추출 우선순위

| 우선순위 | 조건 | 버전 소스 |
|---------|------|----------|
| 1 | K/D/H로 시작하는 파일 + H010 존재 | `H010` 첫줄 앞 3자리 |
| 2 | C/M으로 시작하는 파일 | 파일명 정렬 후 **가장 작은 파일** 첫줄 앞 3자리 |
| 3 | CAR/GHP/REP 확장자 파일 | **해당 파일** 첫줄 앞 3자리 |

---

## 3. SAMFVER_MST 매칭 흐름

```
파일 선택
  ↓
파일명으로 jobssam 결정
  ├─ 확장자 파일 → GHP/CAR/REP
  └─ 비확장자 파일 → 파일명 그대로 (M010.1, K020.2, H010 등)
  ↓
버전 추출
  ├─ K/D/H 파일 → H010 버전
  ├─ C/M 파일 → 최소 파일명 버전
  └─ 확장자 파일 → 자체 첫줄 버전
  ↓
SAMFVER 매칭 (getSamfverMatch.do)
  ├─ 1차: samver + version + valsize → 정확 매칭 (정상)
  └─ 2차: samver + version (valsize 무시) → 길이오류 매칭
  ↓
allTablesDef 조회 (getSamfverAllTables.do)
  └─ 모든 DISTINCT samver+version 조합별로 각각 조회 후 합산
  ↓
모달 카드 렌더링
```

## 4. 파일 세트 예시

### M 계열 (버전: 가장 작은 파일)
```
M010.1  ← 버전 소스 (가장 작은 파일명), SAMVER='M010.1'
M010.2  (빈 파일이면 무시)
M020.1  SAMVER='M020.1'
M020.2  SAMVER='M020.2'
M020.3  SAMVER='M020.3'
M020.4  SAMVER='M020.4'
M020.5  SAMVER='M020.5'
```
