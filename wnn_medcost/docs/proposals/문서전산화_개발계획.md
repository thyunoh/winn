## 목적
종이·팩스로 오가던 컨설팅 문서 5종을 WinCheck+(wnn_medcost) 안으로 옮겨, 병원은 웹에서 작성·동의·제출하고 위너넷은 한 화면에서 관리.

## 대상 문서 · 우선순위

| 단계 | 대상 | 전산화 형태 | 난이도 |
|---|---|---|---|
| **1 (핵심)** | 컨설팅 **의뢰서 + 동의서 2종**(서식1/2/3) | 온라인 접수 폼 + 전자동의 + 관리자 조회 | 중 |
| **1 (핵심)** | 컨설팅 **견적서**(계약서 갈음) | 온라인 견적 발행 + PDF + 이력관리 | 중 |
| 2 | 홍보물·회사소개 | 편집형 홍보 페이지(간이 CMS) | 중 |
| 3 | 프로그램 안내 | 앱 내 도움말·최초 로그인 온보딩 | 하 |
| 3 | 연간 진행 안내문 | 공지 페이지·병원별 일정표 | 하 |

→ **1단계로 실제 수기업무(접수·견적)를 먼저 제거**, 2·3단계는 편집·게시 콘텐츠로 순차 확장.

## DB 설계 (제안 기준)

**① 의뢰서·동의서**
- `TBL_CONSULT_REQ` (의뢰서 마스터, 병원 1건=1행) — req_seq(PK), hosp_cd, hosp_nm, tel, addr, fax, prog_nm, prog_id, **prog_pw_enc**, bed_cnt, **hira_cert_pw_enc**, pc_use_gb, pc_use_time, pc_start_dt, patval_day, goal_score, goal_grade, remark, status(접수/검토/완료), reg_dttm, reg_ip
- `TBL_CONSULT_MANAGER` (담당자, 역할별 N행) — mgr_seq(PK), req_seq(FK), role_gb(총관리자/간호과/심사과/전산), dept, position, tel, email
- `TBL_CONSULT_AGREE` (동의 이력) — agree_seq(PK), req_seq(FK), agree_type(원격접속/개인정보), agree_yn, signer_nm, sign_img, agree_dttm, agree_ip, user_agent

**② 견적서**
- `TBL_QUOTE` — quote_seq(PK), hosp_cd, title, quote_dt, valid_days, supply_amt, vat_amt, total_amt, vat_type, schedule, status(작성/발송/계약), reg_user, reg_dttm
- `TBL_QUOTE_ITEM` — item_seq(PK), quote_seq(FK), no, product, qty, unit_price, amount, remark
- 고정정보 자동삽입: 회사정보(사업자 886-88-00265)·직인·입금계좌(기업 064-173516-01-035 (주)위너넷)

**③ 홍보물** (키-값 블록 CMS)
- `TBL_PROMO_CONTENT` — block_key(PK, 예: hero_title), title, content, img_path, sort_no, use_yn, upd_user, upd_dttm

## 핵심 원칙 (재수정 시 주의)
- **민감정보**(프로그램 PW·심평원 인증서 암호): 저장 최소화 → 부득이 저장 시 **AES-256 + 키 앱과 분리** → 화면 **마스킹**(●●●) → 접근통제·**감사로그** → 전구간 HTTPS.
- **앱 데이터 연계**로 입력 부담 축소: 요양기관기호·병원명은 로그인 정보 **자동채움**, 목표점수·목표등급은 **`TBL_GRADE_MST`**(차등제·연간목표) 값 자동 반영.
- 견적서는 **"계약서 갈음"** 역할.

## 관리자 화면(위너넷)
접수 목록(병원명·기호·접수일·상태 필터/검색) · 상세보기(의뢰서+담당자+동의이력·전자서명) · 상태관리(접수→검토→완료) · 엑셀/PDF 내보내기.

> 스택: **Spring Boot + eGov MVC + MyBatis + JSP/tiles + DataTables + iText(→OpenPDF)**. 출처: `업무기술_웹앱구분.xlsx`.

|---|---|---|---|---|---|
| ① | 의뢰서+동의서 | 웹 | 입력폼 + 담당자 4역할 반복입력 + **전자서명(HTML5 canvas)** + 동의 2종 / Controller·Service·Mapper **트랜잭션**(REQ 1 + MANAGER N + AGREE 2) / 관리자 목록·상세·상태변경 | TBL 3종·AES키 분리·HTTPS·서명이미지 경로 | 2~3일 |

## 기술 전제조건 (착수 전 확인)
- **인프라**: HTTPS 필수 · 배포=**WAR 재빌드→Tomcat 재배포**(반복 사이클이 실제 시간의 큰 변수) · **컨텍스트 경로 확정**(루트 / `/app`) → PWA manifest·SW scope에 직접 영향.
- **DB**: 신규 테이블 **6~7종**(CONSULT_REQ/MANAGER/AGREE · QUOTE/QUOTE_ITEM · PROMO_CONTENT · NOTICE) · PK/시퀀스 정책 기존과 통일.
- **라이브러리**: iText 5.5.13.2(pom 존재) → **OpenPDF 교체 권고(아래 비용 참조)** · NanumGothic TTF 동봉(리눅스 한글 PDF, `classpath:/fonts`) · 멀티파트 업로드(이미 설정).
- **보안**: AES 키 분리보관(프로그램PW·인증서암호, 키는 앱 밖 env/properties) · 접근권한·감사로그·마스킹.
- **앱(PWA)**: `.webmanifest` MIME 매핑(web.xml `application/manifest+json`) · SW 캐시전략(정적 asset만, HTML/.do는 네트워크).
- **재사용**: 로그인·요양기관기호(쿠키) 자동채움·권한 · MagamController·MyBatis·tiles·DataTables·ClientInfo(IP기록) 패턴 복제.

## 비용·라이선스 — 추가비용 없음, 단 PDF 라이브러리 주의
- **결론**: 인프라·서비스 추가비용 **0**(전부 오픈소스 재사용). **유일 주의점 = iText 라이선스.**
- **iText 5 = AGPL/상용 이중** → **상용판매 프로그램은 상용 라이선스 필요 가능** ★
- **→ OpenPDF(iText4 fork, LGPL/MPL, 무료)로 교체 추천** — 상용에 넣어도 무료·소스공개 의무 없음. API 거의 동일(패키지 `com.itextpdf.text`→`com.lowagie.text`, `BaseColor`→`java.awt.Color`), 교체 부담 낮음. Maven `com.github.librepdf:openpdf`. (대안 Apache PDFBox=무료지만 구조 상이해 부담 큼)
- **조건부 유료(원할 때만)**: SMS·알림톡 발송(건당 과금) · 공인전자서명(canvas면 불필요).
