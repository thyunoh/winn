## 목적
종이·팩스로 오가던 컨설팅 문서 5종을 WinCheck+(wnn_medcost) 안으로 옮겨, 병원은 웹에서 작성·동의·제출하고 위너넷은 한 화면에서 관리.

## 대상 문서 · 우선순위

| 단계 | 대상 | 전산화 형태 | 난이도 |
|---|---|---|---|
| **1 (핵심)** | 컨설팅 **의뢰서 + 동의서 2종**(서식1/2/3) | 온라인 접수 폼 + 전자동의 + 관리자 조회 | 중 |
| **1 (핵심)** | 컨설팅 **견적서**(계약서 갈음) | 온라인 견적 발행 + PDF + 이력관리 | 중 |
| 2 | 홍보물·회사소개 | 편집형 홍보 페이지(간이 CMS) | 중 |
| 3 | 프로그램 안내(매뉴얼) | **인터랙티브 매뉴얼**(검색·목차·아코디언·문맥 `?`도움말) + 최초 로그인 온보딩 | 하~중 |
| 3 | 연간 진행 안내문 | 공지 페이지·병원별 일정표 | 하 |

→ **1단계로 실제 수기업무(접수·견적)를 먼저 제거**, 2·3단계는 편집·게시 콘텐츠로 순차 확장.

**① 의뢰서·동의서**

**④ 도움말/매뉴얼** (인터랙티브 매뉴얼 — DB(CMS)로 관리, ③ 홍보물과 동일 패턴)

## 1.2 전자서명(canvas) + 동의 구현 상세

동의 2종([서식2] 원격접속·DB접근 / [서식3] 개인정보 수집·이용)을 **한 화면에서 필수 체크 + canvas 서명 + 증빙(IP·UA·일시) 저장**. 저장은 `TBL_CONSULT_AGREE`에 **동의서별 1행**(원격접속/개인정보), 나머지 컬럼(서명·일시·IP·UA) 구조 공유.

### 서명 입력 방식 — 터치 모니터 불필요 (마우스·터치 겸용)
- mouse + touch 이벤트를 **둘 다** 처리 → **터치 안 되는 데스크탑도 마우스로 서명 가능**(모양은 거침).
- 가장 자연스러운 건 **폰/태블릿 손가락 서명**(PWA 흐름과 일치).
- **결정(#2 연계)**: 터치 없는 접수 PC가 많으면 **"타이핑 서명" 병행**(성명 입력 + 체크 + 접속기록으로 갈음) 권장. → 캔버스 없으면 자동 타이핑 모드로 확장 가능.

### 서명 JS (마우스+터치 겸용 · DPI 대응 · 좌표 스케일)
```javascript
(function(){
  var c = document.getElementById('signPad');
  var ctx = c.getContext('2d');
  var drawing = false, empty = true;
  var dpr = window.devicePixelRatio || 1;              // 선명도(레티나 대응)
  c.width = 600 * dpr; c.height = 200 * dpr; ctx.scale(dpr, dpr);
  ctx.lineWidth = 2.2; ctx.lineCap = 'round'; ctx.strokeStyle = '#111';
  function pos(e){                                       // 화면좌표→캔버스 내부좌표 (CSS크기≠내부해상도)
    var r = c.getBoundingClientRect(); var t = e.touches ? e.touches[0] : e;
    return { x:(t.clientX-r.left)*(600/r.width), y:(t.clientY-r.top)*(200/r.height) };
  }
  function start(e){ drawing=true; var p=pos(e); ctx.beginPath(); ctx.moveTo(p.x,p.y); e.preventDefault(); }
  function move(e){ if(!drawing) return; var p=pos(e); ctx.lineTo(p.x,p.y); ctx.stroke(); empty=false; e.preventDefault(); }
  function end(){ drawing=false; }
  c.addEventListener('mousedown',start); c.addEventListener('mousemove',move); window.addEventListener('mouseup',end);
  c.addEventListener('touchstart',start,{passive:false}); c.addEventListener('touchmove',move,{passive:false}); c.addEventListener('touchend',end);
  window.clearSign=function(){ ctx.clearRect(0,0,c.width,c.height); empty=true; };
  window._signEmpty=function(){ return empty; };
  window._signData=function(){ return c.toDataURL('image/png'); };
})();

function submitConsent(){
  if(!$('#agree_remote').is(':checked') || !$('#agree_privacy').is(':checked')){ alert('두 동의서에 모두 동의해야 합니다.'); return; }
  if($('#signerNm').val().trim()===''){ alert('대표자명을 입력하세요.'); return; }
  if(window._signEmpty()){ alert('서명을 해주세요.'); return; }
  var data = { reqSeq:$('#reqSeq').val(), signerNm:$('#signerNm').val(), signImg:window._signData() };
  CommonUtil.callSyncAjax(CommonUtil.getContextPath()+"/consult/saveAgree.do","POST",data,function(res){
    if(res.IsSucceed){ alert('제출되었습니다.'); location.href=CommonUtil.getContextPath()+"/consult/done.do"; }
    else alert(res.Message||'저장 실패');
  });
}
```

### 서버 — base64→PNG 파일 저장 + 증빙 (eGov Controller)
```java
@RequestMapping(value="/consult/saveAgree.do", method=RequestMethod.POST)
@ResponseBody
public ResponseObject saveAgree(@RequestBody Map<String,Object> p, HttpServletRequest request){
    ResponseObject r = new ResponseObject();
    try {
        String signImg = (String) p.get("signImg");            // data:image/png;base64,xxxx
        byte[] png = Base64.getDecoder().decode(signImg.substring(signImg.indexOf(",")+1));
        String fileName = "sign_" + System.currentTimeMillis() + ".png";
        Files.write(Paths.get(signSaveDir, fileName), png);    // signSaveDir = properties 주입
        String ip = ClientInfo.getRemoteIp(request);           // 기존 유틸 재사용
        String ua = request.getHeader("User-Agent");
        consultService.saveAgree(String.valueOf(p.get("reqSeq")), String.valueOf(p.get("signerNm")),
                                 signSaveDir + "/" + fileName, ip, ua);   // 트랜잭션 2행
        r.setIsSucceed(true);
    } catch(Exception e){ r.setIsSucceed(false); r.setMessage("서명 저장 중 오류"); }
    return r;
}
```

### 구현 주의 4가지 (여기서 잘 막힘)
1. **모바일 터치** — 캔버스 `touch-action:none` + `preventDefault()` 없으면 손가락 그릴 때 화면 스크롤됨.
2. **좌표 스케일** — CSS 크기(width:100%)와 내부 해상도가 달라, `pos()` 비율 변환 안 하면 서명 어긋남.
3. **선명도** — `devicePixelRatio` 배로 내부 해상도 키움(폰에서 안 깨짐).
4. **방어·저장** — 빈 서명·미동의 제출 차단, 서명은 **파일 저장(경로 DB)** 권장(BLOB보다 관리 쉬움).

### 원격-데스크탑 대응 — QR 서명 연계 (토큰·폴링)
원격 + 터치 없는 데스크탑 병원: 의뢰서는 데스크탑에서 입력하고 **서명만 폰으로 넘긴다.** 데스크탑↔폰은 직접 통신하지 않고 **서버가 '토큰'으로 중개** → 같은 망 불필요.

**흐름**
```
데스크탑 "서명하기" → 서버 토큰발급(token ↔ req_seq 저장) + QR(서명URL+token) 표시
   → 폰 QR스캔 → 서명페이지("○○병원 동의") → 손가락 서명 → 제출(token 동봉)
   → 서버가 token으로 req_seq 찾아 AGREE 저장 + token '완료'
   → 데스크탑 폴링(2~3초)으로 완료 감지 → 다음 단계
```
**연결고리 = 토큰 매핑**(`TBL_SIGN_TOKEN`): `token → req_seq → 상태`. 폰은 req_seq 몰라도 token만 들고 오면 서버가 해당 의뢰서를 찾음. (비유: 커피숍 번호표 — PC가 번호표 받아 폰에 넘기고, 서버가 번호표로 주문을 잇는다)

**엔드포인트**
- `/consult/issueSignToken.do` (발급) → token 반환, `token↔req_seq` 저장(만료 10분)
- `/consult/sign.do?token=` (폰 서명페이지) → 만료·위조 방어 + 대상 병원/동의내용 표시, canvas 서명 재사용
- `/consult/saveAgree.do` (token 동봉 저장) → token으로 req_seq 찾아 AGREE 저장 + token '완료'
- `/consult/signStatus.do` (데스크탑 폴링) → `signed` 여부 반환

**데스크탑 JS (QR + 폴링)**
```javascript
CommonUtil.callSyncAjax(ctx+"/consult/issueSignToken.do","POST",{reqSeq:reqSeq},function(res){
  var url = location.origin + ctx + "/consult/sign.do?token=" + res.token;
  new QRCode(document.getElementById("qrBox"), url);          // qrcode.js(무료) 또는 서버 ZXing
  var timer = setInterval(function(){
    CommonUtil.callSyncAjax(ctx+"/consult/signStatus.do","POST",{token:res.token},function(s){
      if(s.signed){ clearInterval(timer); alert("서명 완료!"); location.href=ctx+"/consult/done.do"; }
    });
  }, 2500);
});
```

**보안**: 토큰 **1회용·짧은 만료(10분)** · 서명화면에 대상 병원·동의내용 명시(오서명 방지) · 전구간 HTTPS.
**QR 생성**: 클라이언트 `qrcode.js`(URL만 넣으면 됨) 또는 서버 ZXing(`com.google.zxing`, PWA install-qr 방식).
**실시간화(선택)**: 폴링 대신 SSE/WebSocket으로 즉시 감지 가능(폴링으로 충분).

### 서명 본인확인 강도 — 토큰은 '매칭'만, '신원'은 별도 (핵심 결정)
토큰은 **어느 의뢰서인지(매칭)** 만 보장하고 **누가 서명했는지(신원)** 는 보장하지 않는다(QR 스캔한 사람 아무나 서명 가능). 필요한 본인확인 강도에 따라 선택:

| 강도 | 방법 | 본인확인 | 비용 |
|---|---|---|---|
| **약(기본)** | 대표자명 입력 + 증빙(시간·IP·UA) | 자기신고 | 무료 (현 설계) |
| **중** | 담당자 번호로 **문자 서명링크** 전송 → 그 번호 폰으로만 서명 | "번호 소유자"로 좁힘 | SMS 과금(SENS 재사용) |
| **강** | 휴대폰 본인인증 / 공인·인증 전자서명 | 신원 확인 | 유료·외부 연동 |

- B2B 컨설팅 동의는 대개 **약(이름+증빙)으로 충분**(병원이 알아서 대표가 서명). 분쟁·감사 리스크 크면 중/강.
- **원격일 땐 '중(문자링크)'** 로 한 단계 보강 권장. QR 자체 방어: **1회용·짧은 만료(10분)** + 서명화면에 대상 병원 명시.

### 방문/원격 & 서명링크 전달 (전화번호 필요 여부)
- **방문**: 위너넷 태블릿 지참 → 현장 손가락 서명(canvas). **전화번호 불필요.**
- **원격 + 그 자리**: 화면 **QR 스캔**(본인 폰). **전화번호 불필요.**
- **원격 + 서명자 부재**: **문자·카카오·이메일로 서명링크 전송** → **전화번호(또는 이메일) 필요**. 문자는 SENS(`sendSensApi.do`) 재사용.
- **데스크탑(터치X)**: QR로 폰 서명 or **타이핑 서명** 병행.
- 참고: 의뢰서에 담당자 전화번호를 어차피 받으므로 "문자 서명링크" 옵션 추가 용이.

### 폰 서명 화면 구성 (QR 연계의 `sign.do` 페이지)
**동의문 표시 + 필수 동의 체크 + canvas 서명 + [제출]**. 저장 시 token 동봉 → 서버가 req_seq 매칭 + token '완료'. 사용자 체감: **스캔 → (동의체크) → 사인 → 저장**, 앱 설치 불필요.
- **결정**: 동의 체크를 **폰에서 서명과 함께**(권장 — 서명자=동의자 일치로 증빙 깔끔) vs 데스크탑에서 미리 받고 폰은 서명만.

### 라이브러리 대안
[signature_pad.js](https://github.com/szimek/signature_pad)(MIT, 무료) 사용 시 좌표·터치·DPI 처리 내장 → 더 간단. 위처럼 **의존성 없이 자체 구현도 충분**.

## 기술 전제조건 (착수 전 확인)
- **인프라**: HTTPS 필수 · 배포=**WAR 재빌드→Tomcat 재배포**(반복 사이클이 실제 시간의 큰 변수) · **컨텍스트 경로 확정**(루트 / `/app`) → PWA manifest·SW scope에 직접 영향.
- **DB**: 신규 테이블 **6~7종**(CONSULT_REQ/MANAGER/AGREE · QUOTE/QUOTE_ITEM · PROMO_CONTENT · NOTICE) · PK/시퀀스 정책 기존과 통일.
- **라이브러리**: iText 5.5.13.2(pom 존재) → **OpenPDF 교체 권고(아래 비용 참조)** · NanumGothic TTF 동봉(리눅스 한글 PDF, `classpath:/fonts`) · 멀티파트 업로드(이미 설정).
- **보안**: AES 키 분리보관(프로그램PW·인증서암호, 키는 앱 밖 env/properties) · 접근권한·감사로그·마스킹.
- **앱(PWA)**: `.webmanifest` MIME 매핑(web.xml `application/manifest+json`) · SW 캐시전략(정적 asset만, HTML/.do는 네트워크).
- **재사용**: 로그인·요양기관기호(쿠키) 자동채움·권한 · MagamController·MyBatis·tiles·DataTables·ClientInfo(IP기록) 패턴 복제.

## 비용·라이선스 — 추가비용 없음, 단 PDF 라이브러리 주의
- **결론**: 인프라·서비스 추가비용 **0**(전부 오픈소스 재사용). **유일 주의점 = iText 라이선스.**
- **iText 5 = AGPL/상용 이중** → **상용판매 프로그램은 상용 라이선스 필요 가능** ★
- **→ OpenPDF(iText4 fork, LGPL/MPL, 무료)로 교체 추천** — 상용에 넣어도 무료·소스공개 의무 없음. API 거의 동일(패키지 `com.itextpdf.text`→`com.lowagie.text`, `BaseColor`→`java.awt.Color`), 교체 부담 낮음. Maven `com.github.librepdf:openpdf`. (대안 Apache PDFBox=무료지만 구조 상이해 부담 큼)
- **조건부 유료(원할 때만)**: SMS·알림톡 발송(건당 과금) · 공인전자서명(canvas면 불필요).
