# 월보고서 표준문구(TBL_EVAL_REPORT_TPL) 사용법

> 대상: 위너넷 관리자 · 작성일 2026-07-15
> 관련: 시드 `docs/sql/TBL_EVAL_REPORT_TPL_seed.sql` · 문구 패턴 근거 `docs/reports/월간보고서_총평_작성가이드.md`

---

## 1. 이 테이블이 하는 일

월보고서(적정성평가 월간 컨설팅 보고서) 화면의 **기본 문구를 DB에서 관리**합니다.
문구를 고칠 때 **프로그램 수정·WAR 배포가 필요 없고**, 행을 수정하면 **다음 조회부터 전 병원에 즉시 반영**됩니다.

**우선순위 (같은 위치에 여러 문구가 있을 때 누가 이기나):**

| 순위 | 소스 | 설명 |
|---|---|---|
| ① | 병원별 편집 저장분 | 직원이 화면에서 편집→저장한 문구. 그 병원·그 월에만 적용, 항상 최우선 |
| ② | **TBL_EVAL_REPORT_TPL** (USE_YN='Y') | 전 병원 공통 기본 문구 — **이 문서의 관리 대상** |
| ③ | 프로그램 내장 기본값 | 테이블에 없거나 USE_YN='N'이면 자동 폴백 (화면 자동 초안 포함) |

## 2. 키(SECT_KEY) 체계 — 예약어

SECT_KEY는 화면과 약속된 이름입니다. **이름을 바꾸면 연결이 끊어져 무시됩니다.**

### 지표코드 매핑 (def_XX·dir_XX 공통)

| 코드 | 지표 | 코드 | 지표 |
|---|---|---|---|
| 01 | 의사 1인당 환자수 | 09 | 욕창처치 실시 환자분율 |
| 02 | 간호사 1인당 환자수 | 10 | 욕창이 새로 생긴 환자분율 |
| 03 | 간호인력 1인당 환자수 | 11 | 욕창개선 환자분율 |
| 04 | 약사 재직일수율 | 12 | 일상생활수행능력(ADL) 개선 |
| 05 | 유치도뇨관 보유 환자분율 | 13 | 당뇨환자 HbA1c 적정범위율 |
| 06 | 배뇨관리 실시 환자분율 | 14 | 장기입원(181일 이상) 환자분율 |
| 07 | 항정신성의약품 처방률 | 15 | 지역사회 복귀율 |
| 08 | 의약품(DUR) 점검률 | | |

### 키 그룹별 동작 위치

| 키 | 화면 위치 | 비고 |
|---|---|---|
| `def_01`~`def_15` | Ⅲ. 지표별 분석 박스의 "지표 정의 : …" 줄 | **순수 텍스트만** (HTML·자리표시자 불가) |
| `dir_01`~`dir_15` | Ⅲ "▷ 개선 방향 :" 뒷부분 + Ⅳ 권고 개선방향 | 위와 동일 |
| `sum_p1`~`sum_p5` | 문서 맨 끝 **총평** 문단 1~5 | 자리표시자 사용 가능. 기본 **USE_YN='N'**(화면 자동 초안 우선) |
| `footer`, `diag_note`, `cover_top` 등 | 화면 편집영역(data-key)과 같은 이름이면 그 칸의 기본값 | 자리표시자 사용 가능 |

- 없는 키(예: `def_16`, 임의 이름)를 넣어도 **오류 없이 무시**됩니다. 새 문구 "위치"가 필요하면 화면 개발이 선행돼야 합니다.

## 3. 자리표시자 (sum_p*·화면 편집키 전용 — def/dir에서는 치환 안 됨)

| 자리표시자 | 치환 값 | 예 |
|---|---|---|
| `{total}` | 종합점수 | 87.7 |
| `{grade}` | 현재 등급 | 1등급 |
| `{struct}` / `{care}` | 구조영역 / 진료영역 점수 | 25.1 / 62.6 |
| `{goalGrade}` / `{goalScore}` | 목표등급 / 목표점수 (차등제 등록값) | 1등급 / 90 |
| `{gap}` | 목표점수 − 현재점수 (0 미만이면 0) | 2.3 |
| `{hosp}` | 병원명 | 다솜요양병원 |
| `{ym}` | 평가년월 | 2026년 7월 |

## 4. 기본 사용법

### ① 공통 문구 고치기 (가장 흔한 작업)
```sql
UPDATE TBL_EVAL_REPORT_TPL
   SET TPL_CONTENT = '유치도뇨관(Foley)을 보유한 환자 비율. 값이 낮을수록 우수(0.5% 미만 = 5구간). (2027년 개정 기준)',
       UPD_USER = '홍길동'
 WHERE SECT_KEY = 'def_05';
```
→ 저장 후 월보고서 **재조회**만 하면 전 병원 반영. 배포·재시작 불필요.
(DB 툴에서 그리드 직접 수정해도 동일)

### ② 문구 임시로 끄기 (지우지 않고)
```sql
UPDATE TBL_EVAL_REPORT_TPL SET USE_YN = 'N' WHERE SECT_KEY = 'dir_05';
```
→ 프로그램 내장 기본 문구로 폴백. 다시 켜려면 'Y'.

### ③ 총평을 자동 초안 대신 고정 문구로
기본은 화면이 병원 수치로 만든 **자동 초안(구어체)**이 채워집니다.
특정 문단을 전 병원 동일 문구로 고정하려면:
```sql
UPDATE TBL_EVAL_REPORT_TPL
   SET USE_YN = 'Y',
       TPL_CONTENT = '{ym} 예상 종합점수는 {total}점({grade})입니다. 목표 {goalGrade}({goalScore}점)까지 {gap}점 남았습니다.'
 WHERE SECT_KEY = 'sum_p1';
```
→ 자동 초안 대신 이 문구(수치 치환)가 들어감. 되돌리려면 USE_YN='N'.

### ④ 병원별 예외 문구
DB가 아니라 **화면에서**: 해당 병원 월보고서 → 편집 켜기 → 문구 수정 → 저장.
그 병원·그 월만 바뀌고(우선순위 ①), 공통 문구는 그대로입니다.
예외를 없애고 공통으로 되돌리려면 편집으로 원래 문구를 복원해 저장하거나, 해당 월 데이터를 재저장.

### ⑤ 시드 다시 실행해도 되나?
`TBL_EVAL_REPORT_TPL_seed.sql`은 `INSERT IGNORE`라 **이미 있는 행은 건드리지 않습니다.**
실수로 행을 삭제했을 때 재실행하면 삭제된 행만 초기값으로 복구됩니다.
전체 초기화는 `DELETE FROM TBL_EVAL_REPORT_TPL;` 후 재실행 (수정분도 사라지니 주의).

## 5. 주의사항 (꼭 지킬 것)

1. **SECT_KEY 이름 변경 금지** — 화면과의 연결 키. 바꾸는 순간 무시됩니다.
2. **def_/dir_에는 HTML 태그·자리표시자 금지** — 순수 텍스트만 (태그를 넣으면 `<b>` 글자가 그대로 보임).
3. **새 키 INSERT는 의미 없음** — 화면이 아는 키는 이미 전부 시드됨.
4. 총평 sum_p*를 'Y'로 켜면 **모든 병원**이 같은 문구가 됩니다 — 병원 수치 기반 자동 초안이 사라지므로, 켜기 전에 자리표시자로 수치를 넣었는지 확인.
5. 문구는 **다음 조회부터** 반영 — 이미 열어둔 화면은 재조회(조회 버튼) 필요.
6. 병원별로 이미 편집·저장된 문단은 공통 문구를 바꿔도 안 바뀝니다(우선순위 ①이 이김).

## 6. 자주 하는 시나리오 요약

| 하고 싶은 것 | 방법 |
|---|---|
| 심평원 기준 개정으로 지표 설명 교체 | `def_XX` TPL_CONTENT 수정 |
| 개선방향 권고 문구 톤 변경(예: 구어체) | `dir_XX` TPL_CONTENT 수정 |
| 총평 특정 문단 전사 통일 | `sum_pN` 내용 작성 + USE_YN='Y' |
| 총평 자동 초안으로 복귀 | `sum_pN` USE_YN='N' |
| 특정 병원만 다른 문구 | 화면에서 편집→저장 (DB 작업 아님) |
| 문구가 이상해져 원래대로 | 해당 행 USE_YN='N' → 내장 기본값 폴백 |
